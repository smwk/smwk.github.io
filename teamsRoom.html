Teams not showing Rooms
25-Mar-23

Issue: when we go to make a new meeting within the Teams client we find that no rooms are visible. Additionally we can only add a Room as an attendee by entering the full email address, there is no address suggestion displayed.

↑ Error message: No rooms found. Try another name or room list.


The Rooms List, which works fine in Outlook, also shows no rooms within Teams. If we try to add a migrated room to a Rooms List we get an error: Only room mailbox or room list can be added into room list.

↑ Unable to add an O365 Room mailbox from on-prem Exchange PowerShell.


Troubleshooting: when a mailbox (or Room) is migrated to O365 then the recipientTypeDetails property is set to MailUser. So this would explain why there is an error but so far I've found no resolution to adding the object to the room list other than adding them to the list before migration.

For the issue with the rooms not displaying in the Location field in Teams I have tried adding additional information to the mailbox using on-prem commands that will sync to O365 fully after about 24 hours. Despite this the rooms still do not appear.

While researching this issue I did not find anyone with the same problems, so decided to open a support call with Microsoft, which is currently ongoing.
