Delegate Sent Items placed into external Tenant
12-Mar-23

Senario
You give a user the 'Send As' permission to another user's mailbox and then enable the 'Automatically save sent items in delegator's mailbox in Exchange Online'. The sent items appear in both mailboxes as expected. However you also forward all mails from this mailbox to an external mailbox not under your control. The user notices that the sent item also appears in this external mailbox.

Cause
This 'functionality' has been described before where sent items would appear for members of a distribution group if the delegee automatically forwarded to that group. But even then it was still a big surprise to learn this would work across domains. The forwarded exrternal mailbox also resides in O365, and possibly in the same datacenter, so that may explain why it occurred, though I never got an opportunity to test all scenarios. Assuming Microsoft classify it as a bug then it doesn't look like it will be patched anytime soon.

Resolution
There seemed to be 3 options, 1 accept this was the functionality, 2 disable the save sent items option, 3 use webmail to directly logon to the delegee's mailbox and send from there. For a few reasons it was decided option 3 would be best.
